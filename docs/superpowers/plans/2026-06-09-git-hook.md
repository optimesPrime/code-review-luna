# Git Hook 一键安装 — luna 作为提交守卫

**目标：** `luna install-hook` 把 Luna 安装为 git pre-commit 或 pre-push hook。每次提交/推送前静默运行，只在有阻塞级风险时拦截并打印摘要，否则放行。

---

## 产品行为

```bash
# 安装 pre-commit hook（默认）
luna install-hook

# 安装 pre-push hook
luna install-hook --hook pre-push

# 卸载
luna uninstall-hook

# hook 触发时的终端输出（只在阻塞时）
$ git commit -m "add trade header"

🌙 Luna Review · pre-commit
🚫 阻塞提交 · 🚨 2 高风险

  请求上下文  X-Trade-UserId 首个请求可能缺失   request.ts:42
  权限/登录态  新增接口缺少 [Authorize] 标注     api.ts:88

运行 luna 查看完整结果，或加 --no-verify 跳过审查。
```

无高风险时静默放行，不打印任何内容。

---

## 架构

```
luna install-hook
  └── hook_installer.py
        ├── 写 .git/hooks/pre-commit（shell 脚本）
        └── chmod +x

pre-commit hook 脚本内容：
  luna --staged --quiet --format json 2>/dev/null | \
  python3 -c "import sys,json; r=json.load(sys.stdin); sys.exit(1 if r.get('verdict')=='blocked' else 0)"
```

实际上 hook 脚本直接调 `luna --staged --hook` 内置 flag，Luna 自己决定 exit code，不依赖外部 jq / python 管道：

```bash
#!/bin/sh
luna --staged --hook --config ~/.luna/config.yaml
```

`--hook` flag 让 Luna 运行在 hook 模式：
- 只输出摘要（等同 `--quiet`）
- 只有 verdict=blocked 时 exit(1)，其余情况 exit(0)
- 不写报告文件（避免每次提交都生成报告）

## Task 4：验证

```bash
pytest -q
python3 luna.py install-hook --help
python3 luna.py uninstall-hook --help
```

手动验证（在一个测试 git 仓库中）：

```bash
luna install-hook
cat .git/hooks/pre-commit
git add . && git commit -m "test"   # 无高风险时静默放行
luna uninstall-hook
cat .git/hooks/pre-commit           # 应该不存在
```

验收标准：
- 无高风险时 git commit 静默放行
- 有阻塞级风险时打印精简摘要，git commit 被拦截
- 手动 `--no-verify` 可绕过（标准 git 行为）
- 已有非 Luna hook 时不覆盖，打印警告

## Non-Goals（本阶段不做）

- pre-push 时审查完整 branch diff（只审查 staged 或 HEAD）
- 团队统一分发 hook（通过 git config core.hooksPath）
- Windows 支持（PowerShell hook 脚本）
