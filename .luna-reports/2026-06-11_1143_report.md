## 三、爆炸范围分析

### `build_graph` → `phases/context_graph.py:53`
- 风险: **high** | 置信度: medium *(需人工确认)*
- 原因: build_graph 引入 SQLite 增量缓存后，未变更文件会跳过解析，只恢复 import 边；如果 ContextGraph 还依赖函数、组件、导出等节点信息，缓存命中路径下这些节点不会被重建，可能导致后续 blast radius、符号提取或风险传播漏报。
- 建议: 确认 SQLite 缓存是否持久化并恢复完整的 ContextGraph 节点信息；若仅缓存文件 hash 和 import edges，应避免跳过完整解析，或扩展缓存表保存并恢复 symbols/nodes。

### `build_graph` → `phases/context_graph.py:84`
- 风险: **medium** | 置信度: high
- 原因: 缓存命中时直接 continue，依赖后续从 DB 恢复边关系；一旦 DB 中 edges 与文件 hash 状态不一致，内存图会缺失未变更文件的依赖关系，影响 graph.find_usages 和风险传播结果。
- 建议: 为缓存 schema 增加版本号或一致性校验；当发现 edges 缺失、schema 变更或缓存异常时，强制全量重建并重写缓存。

### `build_graph` → `phases/context_graph.py:111`
- 风险: **medium** | 置信度: medium
- 原因: 同步 SQLite 时直接访问 db._conn 私有连接，并吞掉所有异常；缓存写入或删除失败不会暴露，可能长期使用脏缓存，导致影响链路不完整。
- 建议: 为 GraphDB 增加公开的事务、删除和 upsert API；至少记录异常日志，并在写入失败时标记缓存不可用或触发下次全量重建。

### `build_graph` → `phases/context_graph.py:134`
- 风险: **medium** | 置信度: high
- 原因: 新增缓存约定为 DB 中 source=imported、target=importer，而最终 graph.edges 又转换为 source=importer、target=imported。该双重方向约定容易在后续维护中引入反向边错误，直接影响依赖传播方向。
- 建议: 将边方向约定封装在 GraphDB API 中，并补充单元测试覆盖 A imports B 时 DB 边、graph._importers、graph.edges、find_usages 的方向一致性。

### `propagate_risk` → `phases/risk_propagation.py:65`
- 风险: **medium** | 置信度: medium
- 原因: 风险传播从直接读取 graph._importers 改为调用 graph.find_usages(current)。如果 find_usages 的语义不完全等价于“查找当前文件的 importers”，可能改变传播范围，出现漏报或过度传播。
- 建议: 补充回归测试：给定 A imports B、C imports A，验证从 B 开始传播时结果包含 A 和 C，且最大深度、visited 行为保持不变。

### `cli` → `luna.py:278`
- 风险: **low** | 置信度: high
- 原因: CLI 移除了 context-graph.json 的显式加载与保存，改为每次调用 build_graph，由 build_graph 内部处理 SQLite 增量缓存。主要影响是缓存格式和缓存路径切换，可能影响首次运行性能和旧缓存兼容性。
- 建议: 确认不再需要兼容 .luna/cache/context-graph.json；如需平滑迁移，可在首次 SQLite 缓存不存在时允许读取旧 JSON 缓存或清理旧缓存文件。


## 四、代码质量问题

### `phases/context_graph.py:94` — 命中 SQLite 缓存的文件会直接跳过解析，但后续只从数据库恢复 import 边关系，没有恢复该文件解析产生的符号节点、导出信息等完整图内容，导致增量缓存命中时生成的 ContextGraph 不完整。
- 类型: logic_gap | 风险: **high** | 置信度: high
- 依据: if db is not None and cached_hashes.get(str(src)) == new_hash:
    continue  # unchanged — skip tree-sitter parsing
...
for imported_abs, importer_abs in db.get_all_edges():
    ...
    graph._importers[imported_rel].add(importer_rel)
- 建议: 要么 SQLite 缓存完整持久化并恢复 nodes/edges/符号信息，要么即使文件 hash 未变化也至少恢复该文件的完整图数据；在未实现完整恢复前，不应直接跳过解析。

### `phases/context_graph.py:157` — 使用缓存跳过部分文件解析后，SQLite 同步或读取全部边关系阶段如果发生异常会被直接吞掉，函数继续返回不完整的内存图，且没有触发全量重建兜底。
- 类型: missing_error_handling | 风险: **high** | 置信度: high
- 依据: if db is not None and cached_hashes.get(str(src)) == new_hash:

### `phases/context_graph.py:78` — 扫描源码文件阶段捕获 ValueError 后直接忽略，会导致 source_files 保持空列表并返回空图，缺少全量失败提示或降级逻辑。
